Tutorial: Getting help

Similar to standard MATLAB, you can obtain help for any function, class or method in the Computational Physiology Toolbox using the help function at the command line.

Help for functions

The following code displays the help text on the function that does lean body weight-based scaling.
help scaling_LBW
scaling_LBW Lean body weight-based scaling of physiological parameters. TARGPHYS = scaling_LBW(ORIGPHYS, TARGCOV), with Physiology objects ORIGPHYS and TARGCOV, scales the detailed physiological properties of ORIGPHYS to the covariate information in TARGCOV using the scaling approach described in Huisinga et al., CPT:PSP (2012) 1:e4. This scaling method is only applicable for humans. Physiological parameters required in ORIGPHYS: 'species','sex','age','BH','BW','BSA','hct','dens','rtpAlb', 'rtpLip','pH','fqbloCO','fnliVtis','fnphVtis','faphVtis', 'fintVtis','fwecVtis','fwicVtis','fwtotVtis','fregVtbl','fartVtbl', 'fvenVtbl','OWtis','Vtbl','fcelVtis','co'. Possible physiological parameters in TARGCOV: 'species','sex','age','BH','BW'. If any of these parameters are not specified in TARGCOV, they will be taken from ORIGPHYS instead. Output TARGPHYS will have the physiological parameters required in ORIGPHYS, scaled to TARGCOV. Examples: origPhys = Physiology('human35m'); targCov = Covariates('Species','human', ... 'sex', 'male', ... 'age', 40*u.y, ... 'BW', 70*u.kg, ... 'BH', 1.70*u.m); targPhys = scaling_LBW(origPhys, targCov); See also scaling_BW, Physiology, Covariates
Following MATLAB conventions, input/output arguments to the function described are shown in capital letters. At the bottom, there are links to related help pages, in this case, to another scaling method and to functions Physiology() and Covariates() which create physiologies.

Help for classes

For classes like Bolus, the help usually displays the purpose of the class:
help Bolus
Bolus Class for storing bolus dosing events See also Dosing, SimpleDosing, ComplexDosing, Oral, Infusion, Bolus/Bolus (constructor). Documentation for Bolus
However, it does not explain how to create an object of that class. For that information, type
help Bolus/Bolus
Bolus Create a dosing object with bolus dosing OBJ = Bolus(COMPOUND, TIME, DOSE, TARGET) creates an object of class Bolus, OBJ, containing information from bolus dosing. The following input is expected: * COMPOUND: a character array (e.g. 'Warfarin') or a cellstr * TIME: convertible to a Time-type DimVar, e.g. 0*u.h or '0 h' * DOSE: in most cases, a Mass- or Amount-type DimVar, e.g. 200*u.ug or '200 ug', but units are not checked to allow for dosing e.g. in [mg/kg] * TARGET: a character array (e.g. 'iv') or a cellstr OBJ = Bolus(TAB) with a table TAB containing columns 'Compound', 'Time', 'Dose' and 'Target' as specified above, is an equivalent way of specifying bolus dosing. Examples: Bolus('Warfarin',0*u.h,10*u.mg,'iv') See also Infusion, Oral, ComplexDosing Documentation for Bolus/Bolus
In addition, by clicking on "Documentation for Bolus" below the class help provides additional information on the class properties and methods. For example, you can see the Bolus class has one properties: schedule (the tabular data structure in which bolus dosing information is stored), and seven methods: combine(), compounds(), disp(), dosingTypes(), filterDosing(), plus(), and summary().
You can click on one of these to get help, say method compounds. The help page can also be accessed from the console via
help Bolus/compounds
compounds Return compounds defined in a SimpleDosing object CPD = compounds(D), with a SimpleDosing object D, returns the compounds specified in the schedule as a cellstr CPD. Help for Bolus/compounds is inherited from superclass SimpleDosing

Function about()

Next, in addition to the help command, there is a specific function about() which provides information on physiological or drug-related parameters. This function can be used in different ways.
First, you can get a summary of the database content. As an example, we query the 250g reference rat in the physiological database:
rat = Physiology('rat250');
about(rat)

and you get a summary of records stored in this database.
Second, for any defined physiological or drug-related parameter in a database, you can get information about its meaning. For example, you might wonder what is fwicVtis in the summary above. Querying this parameter
about('fwicVtis')
Description: Intracellular water fraction of tissue organ volume Unit type: unitless Per tissue parameter
gives the answer: Intracellular water fraction of tissue organ volume. It also provides the unit type of the parameter, in this case, it's unitless, and tells you whether the physiological parameter is scalar or defined per tissue.
Combining the two above calls, you can move one level into the database:

Notation

The final resource for getting help is the tutorial "Notation", which contains lists of important abbreviations used consistently within the toolbox:
  • abbreviations for tissue subspaces, such as "cel" for the cellular part of a tissue,
  • abbreviations for organs, like "adi" for adipose tissue,
  • and finally abbreviations for tissue constituents, which are used during prediction of tissue partitioning.
