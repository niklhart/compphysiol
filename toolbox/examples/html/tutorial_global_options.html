Tutorial: Global options

Global options in the Computational Physiology Toolbox are used for different purposes, for example to control the default behaviour of plotting functions. To see the current status of these options, type
getoptcompphysiol
ans = struct with fields:
reporting: 'Assumptions' XScaleLog: 0 YScaleLog: 0 OdeUnitCheck: 'once' DisplayUnits: {} AutoAssignDrugData: 0 AutoFilterDrugData: 0 AutoExpDataUnitType: 1 PlotOptions: [1×1 struct] LoadExpDrugData: 1 DefaultObservable: [0×0 Observable] ReportToConsole: 0 DimVarPlot: 0 PhysiologyDB: @initphysiologydb DrugDB: @initdrugdb PhysiologyTemplate: @physiologytemplate DrugTemplate: @drugtemplate PlotTemplate: @plottemplate ObservableTemplate: @obstemplate
These are the default options at startup, for example, no custom dispay units are used and plots are not shown on logarithmic scale.
To know more about these options, you can consult the help of the optionsClass.
help optionsClass

Accessing a global option

Global options are accessed in many toolbox functions, for example, if not specified otherwise, plotting functions use linear or logarithmic plots depending on the global options. Also, many functions use the global options to access the physiological or drug databases. During scripting, it is often not needed to access global options directly, but in principle, this is done by calling getoptcompphysiol() with an input:
getoptcompphysiol('PlotOptions')
ans = struct with fields:
lw: 2 ms: 10 fs: 18

Changing a global option

To change a global option, use function setoptcompphysiol(). As an example, let us set some custom display units. If you type
5*u.cm^3
ans = 5.0000e-06 m^3
you get the result in cubic meters, the default. But maybe we rather want to see all volumes in liters. Let's change the display units (and store the previous setting in the variable oldopt):
oldopt = setoptcompphysiol('displayUnits',{'L','nM','min'})
oldopt = struct with fields:
reporting: 'Assumptions' XScaleLog: 0 YScaleLog: 0 OdeUnitCheck: 'once' DisplayUnits: {} AutoAssignDrugData: 0 AutoFilterDrugData: 0 AutoExpDataUnitType: 1 PlotOptions: [1×1 struct] LoadExpDrugData: 1 DefaultObservable: [0×0 Observable] ReportToConsole: 0 DimVarPlot: 0 PhysiologyDB: @initphysiologydb DrugDB: @initdrugdb PhysiologyTemplate: @physiologytemplate DrugTemplate: @drugtemplate PlotTemplate: @plottemplate ObservableTemplate: @obstemplate
We have also set a sensible display unit for molar concentrations and one for times. Now let's do the above computation again:
5*u.cm^3
ans = 0.0050 L
and you see the result is given in liters.

Restoring the previous state of an option

Above, function setoptcompphysiol() was used with a return value, oldopt. This allows to reset the options to their previous state:
setoptcompphysiol(oldopt)
and we're back to cubic meters:
5*u.cm^3
ans = 5.0000e-06 m^3
You can also restore the default options at toolbox startup using the command
clear optionscompphysiol
