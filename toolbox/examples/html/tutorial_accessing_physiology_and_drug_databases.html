Tutorial: Accessing physiology and drug databases

Mechanistic pharmacokinetic modelling relies on physiological and drug related-parameters. In the Computational Physiology Toolbox, the physiology and drug databases are used to store such data in a systematic way. In this tutorial, we will look at the general structure of these databases and how to access data from the databases.
Let's start with the physiology database. This database consists of an array of Physiology objects with detailed physiological information for a set of reference individuals, based on data collected from the literature.
Let's look at this object once:

During modelling tasks, we do not need to access the database in this way, but rather use function Physiology() to access one of these reference physiologies. For this tutorial, we will consider a 35-year old female adult:
human35f = Physiology('human35f');
Let us also consider the drug database, again built from literature data:

Four ways of accessing the drug database

We can also query individual parameter names without referring to any database to get details about these parameters.
about('BW')

about('Vtis')

In particular, we see that body weight is a scalar parameter, whereas Vtis is a per-tissue parameter. These are the two types of physiological parameters, and it is important to keep this distinction in mind.

and we see that there are also two types of drug-related parameters, namely scalar parameters, which are purely physiochemical parameters such as pKa, and species-specific parametes like the gut extraction ratio, which contain some organism-specific pharmacokinetic information.
Function "about" gives a quick overview, however it only prints the database content, which means we cannot use it subsequently in computations. It is meant as a way to explore a database.

Second way of accessing databases: function getvalue()

Next, we can do a formal database query using function getvalue():
getvalue(human35f, 'BW')
ans = 60 kg
which returns the body weight of the reference individual, or
getvalue(human35f, 'Vtis', 'lun')
ans = 420 mL
which returns the lung tissue volume.
However, we cannot simply get all tissue volumes in the database together using this syntax, since the returned vector would not contain any information on which value is which:
getvalue(human35f, 'Vtis')
Error using DB/getvalue
Several matches for parameter "Vtis" found in Physiology object "human35f".
It is possible, however, to obtain specific entries for per-tissue parameters at once, e.g. lung and adipose tissue volumes:
getvalue(human35f, 'Vtis', {'lun' 'adi'})
ans = 1.0e+04 * 0.0420 2.0742 mL
We can do the same queries for the drug database:
getvalue(lidocaine, 'MW')
ans = 0.2343 kg --- mol
getvalue(lidocaine, 'Egut', 'human')
ans = 0

Third way of accessing databases: function loaddatabases()

Function getvalue() is best suited for computations at the script level.
However, within mechanistic models, there are many queries of a single database for a fixed set of tissues, like the 12 tissues in the standard well-stirred 12-compartment-PBPK model. There's a specific function for this setting, called loaddatabases(). This is the third way of accessing a database. It is called in the following way, providing a physiology, drug data and an array of tissues:
[queryphys, querydrug, I] = loaddatabases(human35f, lidocaine, {'lun','adi'})
queryphys = function_handle with value:
@(varargin)getfromphys(phys,cmt,varargin{:})
querydrug = function_handle with value:
@(varargin)getfromdrug(drug,varargin{:})
I = struct with fields:
lun: 1 adi: 2
It returns two new query functions, queryphys() and querydrug(), which are wrappers around function getvalue(), and an indexing struct I.
queryphys('Vtis')
ans = 1.0e+04 * 0.0420 2.0742 mL
can now be used as shorthand for
getvalue(human35f, 'Vtis', {'lun' 'adi'})
ans = 1.0e+04 * 0.0420 2.0742 mL
That is, queries for tissue-dependent parameters are vectorized for all tissues. The indexing struct I can be used to relate the name of the tissue to the index in the vector returned by queryphys(). You can see this principle at work in any of the models.
Also,
querydrug('Egut')
ans = 0
is shorthand for
getvalue(lidocaine, 'Egut')
ans = 0

Fourth way of accessing databases: directly using the database structure

Finally, we can directly use the underlying implementation of the databases. For example,
human35f.db.BW.Value
ans = 60 kg
yields the same result as
getvalue(human35f, 'BW')
ans = 60 kg
However, a direct access to the database should be done with special caution. It does not have any error handling, it doesn't support the vectorized assignments shown above.

Exercise

Here's an exercise to get used to accessing databases. In a 12-compartment whole-body PBPK model, the 10 tissues
adipose tissue, bone, gut, heart, kidney, liver, lung, muscle, skin and spleen,
are modelled, as well as blood (tbl). Which percentage of body weight does the sum of these organ weights constitute? Compare these values for an adult, a child, a rat, and a mouse. Note that tbl is a different type of physiological parameter than the others. Try to do the computation in the toolbox; you can check your result in the solutions.
