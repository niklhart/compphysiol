Tutorial: Defining a physiology

To run a simulation in the Computational Physiology Toolbox, the individual components of a simulation task must be specified. In this tutorial, we will look at how to specify physiological information.

Overview: how to specify a physiology

There are three different ways to define a physiology:
  1. Defining basic covariates at the script level (which is sufficient for empirical PK models).
  2. Access the physiology database and retrieve detailed physiological information on a reference individual. This is the level of detailedness required by PBPK models.
  3. To define detailed physiology for a non-reference individual, you can use a physiological scaling method.
Let's look at how to implement each of these approaches.

Specifying basic covariates

To define basic covariates at the script level, use function Covariates(). For example, let's define a normal male adult:
human1 = Covariates('Species', 'human', 'BW', 70*u.kg, ...
'BH', 1.80*u.m, 'sex', 'male', 'age', 35*u.year)
human1 =
Physiology object: species: human sex: male age: 35 year BH: 1.8 m BW: 70 kg
Function Covariates() allows to define any number of physiological parameters. For example,
Covariates('Species', 'human')
ans =
Physiology object: species: human
is also a valid physiology. To see how per-tissue physiological parameters are specified, refer to the function help.

Specifying a physiology by physiological scaling

The detailed physiology of a reference individual can be scaled to a target individual specified by some covariates. Two scaling methods are implemented, lean-bodyweight-based scaling and bodyweight-based scaling.
As an example, we will scale the detailed physiology human2 to the covariates human1, using lean-bodyweight-based scaling:
human3 = scaling_LBW(human2, human1)

Each scaling method has specific requirements on the physiological parameters defined in the reference and target individual's physiology. These requirements can be found in the documentation of the scaling functions. In the above example, they were met, but otherwise an error will be produced.

Overarching concept: Physiology objects

Note that all three of these methods result in the same kind of object, an object of class Physiology. You can think of this class as comprising a continuum of degrees of detailedness, with very coarse-grained covariate information on the one end, and very detailed physiology at the other end.
